# Web 客户端帖子渲染验收记录（2026-06-14）

## 测试范围

- 客户端：`http://localhost:8081/`
- 官方论坛：`https://bbs.yamibo.com/`
- 浏览器：Chrome（Codex Chrome Extension）
- 账号状态：已登录（`cola43`）
- 仅检查只读 v1 已实现功能；发帖、回复、点赞、收藏写入、搜索等未实现功能不计为缺陷。
- 覆盖板块：动漫区、海域区、贴图区、文学区、游戏区、影视区、资源交流区。
- 重点检查：帖子列表数据、正文与回复、Discuz 富文本、图片、图片查看器、正文链接、多页回复。

## 结论

普通纯文本帖的标题、作者、时间、正文和首批回复基本可以正常显示。主要问题集中在富文本解析：Discuz 格式控制串泄漏、URL 不可点击、部分图片只显示“图片”占位或加载失败、竖图被严重裁切，以及回复数较多的帖子只加载一部分回复却显示“到底啦”。

## 发现的问题

### P1：Discuz 格式控制串大量泄漏到正文和引用

**影响**

正文被随机字母、数字、标点和反斜杠打断，部分帖子接近不可读；回复引用中也会出现同类内容。

**复现帖子**

- 贴图区：`[街角魔族] [动图] 夏美子和桃`
- 贴图区：`《毁掉一切地狱再爱》封面1-6`

**实际表现**

- 正文出现 `7 ~/ a( I3 h$ r' x5 f3 _`、`; E; c$ h6 S- d2 R`、`N9 T- C) \\8 \\7` 等无意义片段。
- 回复引用也会混入类似 `# l) a# Y0 m2 r/ Q5 p$ w9 O` 的内容。
- 同一帖的列表摘要已经清理过这些内容，进入详情后反而出现污染。

**期望**

与官方论坛一致，只显示用户可见正文；Discuz 的内部格式/干扰串不应进入客户端文本。

---

### P1：正文中的 URL 全部不可点击

**影响**

资源区、影视区、游戏区和海域区大量依赖网盘、视频、站内专楼链接，目前只能手动抄写，核心只读浏览体验不可用。

**复现帖子**

- 海域区：`该怎么办!`，正文包含 `https://b23.tv/kpi2xtH`
- 贴图区：`[街角魔族] [动图] 夏美子和桃`，包含百度盘和 MEGA
- 游戏区：`[NS2]ファイアーエムブレム 万紫千紅 火纹新作2026`，包含 YouTube 和 Nintendo
- 影视区：`热播中的百合剧整合（5月时间表更新）`，包含大量站内专楼和微博链接
- 资源交流区：`[自购][角川][羽田宇佐&右腹]我买下了与她的每周密会/周次 卷1`，包含夸克网盘

**实际表现**

- URL 与普通正文使用相同样式。
- 点击 URL 后页面、地址和 Chrome 标签均无变化。
- 页面 DOM 中没有正文 `<a>` 元素。
- HTML 实体未完全解码，影视整合帖可见 `&amp;fromuid=484118`。

**期望**

识别 `http://`、`https://`、站内主题链接和 BBCode URL，提供明确的链接样式并可点击打开。

---

### P1：部分帖子图片只显示“图片”，实际图片未渲染

**复现帖子**

- 游戏区：`[NS2]ファイアーエムブレム 万紫千紅 火纹新作2026`
- 影视区：`热播中的百合剧整合（5月时间表更新）`

**实际表现**

- 火纹帖子首楼连续显示 8 个“图片”文本占位，但页面中除头像外没有对应正文图片。
- 火纹帖回复楼层也有相同“图片”占位。
- 影视整合帖的时间表显示“图片”；检测到的新浪外链图片 `naturalWidth/naturalHeight` 均为 `0`，加载失败。

**期望**

官方论坛可见的附件、BBCode 图片和允许显示的外链图片应正常渲染；失败时至少显示可理解的失败状态或原图链接，而不是无交互的“图片”二字。

---

### P1：竖版图片按固定横向比例裁切，无法在正文中看全

**复现帖子**

- 贴图区：`《毁掉一切地狱再爱》封面1-6`

**实际表现**

- 原图尺寸多为约 `844 × 1200`，正文统一渲染为约 `331 × 221`。
- 图片上下大面积内容被裁掉，只保留中间横向区域。
- 多张封面连续显示时均存在相同问题。

**期望**

正文图片应按原始宽高比等比缩放，完整显示；可限制最大高度，但不应默认使用横向 `cover` 裁切。

**补充**

点击图片可进入大图查看器，大图查看器能显示完整图片；该功能本身正常。

---

### P1：多页主题只显示首批回复，却错误显示“到底啦”

**复现帖子**

- 动漫区：`在看漫画时大家更喜欢“我喜欢你”还是“请爱着我”`
  - 页面显示 `115 条回复`
  - 实际只显示到 `#20`
- 文学区：`漩涡——被迫驻足不前的我们（2026.3.31更新最终话，附txt）`
  - 页面显示 `110 条回复`
  - 当前详情仅加载首批楼层
- 影视区：`热播中的百合剧整合（5月时间表更新）`
  - 页面显示 `96 条回复`
  - 当前详情显示首楼、`#102`，随后是 `#2` 至 `#20`，底部直接显示“到底啦”

**实际表现**

- 没有加载更多入口或继续加载行为。
- 已加载回复排序异常：影视整合帖把最新的 `#102` 插在首楼后，再从 `#2` 开始。
- 底部“到底啦”会误导用户认为已看到全部回复。

**期望**

支持主题回复分页/加载更多；未加载完时不得显示“到底啦”。回复排序应明确且连续。

---

### P2：正文图片统一固定高度，横图也可能发生不必要裁切

**复现帖子**

- 贴图区：`[街角魔族] [动图] 夏美子和桃`

**实际表现**

- GIF 可以加载并播放，点击后也能进入 `1 / 1` 图片查看器。
- 正文图片区域固定为约 `331 × 221`；该图接近横图所以影响较轻，但与竖图问题表明组件没有按原图比例布局。

**期望**

所有正文图片统一使用等比缩放策略，不依赖固定高度。

## 测试限制

Chrome 可以识别并接管已打开的官方论坛标签页，但对该标签页执行 DOM 读取、截图或导航时持续超时；因此本轮对照主要依据客户端中的官方实时数据、帖子元数据、附件 URL 和可见内容完成，未能对每个抽样主题逐项截取官方页面作为视觉证据。该限制不影响 URL 无交互、客户端 DOM 无链接、图片自然尺寸为 0、竖图裁切、回复数量不完整和格式控制串泄漏等客户端侧问题的复现结论。
